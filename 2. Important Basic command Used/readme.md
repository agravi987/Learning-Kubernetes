# Kubernetes Basic Commands Documentation

## 1. Cluster Information Commands
These commands help verify that the cluster is running and accessible.

### Check Cluster Information
**Command:**
```bash
kubectl cluster-info
```

**Purpose:**
Displays information about the Kubernetes control plane and core services.

**Expected Output:**
```text
Kubernetes control plane is running at https://127.0.0.1:6443
CoreDNS is running at https://127.0.0.1:6443/api/v1/namespaces/kube-system/services/kube-dns:dns/proxy
```

**Use Case:**
Used to confirm that the cluster is accessible.

---

### View Cluster Nodes
**Command:**
```bash
kubectl get nodes
```

**Purpose:**
Lists all nodes participating in the cluster.

**Expected Output:**
```text
NAME        STATUS   ROLES           AGE   VERSION
minikube    Ready    control-plane   5h    v1.29
```

**Explanation:**

| Column | Description |
| :--- | :--- |
| **NAME** | Node hostname |
| **STATUS** | Node health |
| **ROLES** | Worker or control plane |
| **AGE** | Node uptime |
| **VERSION** | Kubernetes version |

---

## 2. Pod Management Commands
Pods are the smallest deployable unit in Kubernetes.

### List Pods
**Command:**
```bash
kubectl get pods
```

**Purpose:**
Displays all pods running in the current namespace.

**Expected Output:**
```text
NAME                       READY   STATUS    RESTARTS   AGE
nginx-6d4cf56db6-hk2kp     1/1     Running   0          5m
```

---

### Detailed Pod Information
**Command:**
```bash
kubectl describe pod <pod-name>
```

**Purpose:**
Displays detailed information about a pod.

**Information Provided:**
- Container image
- Events
- Node placement
- Network settings
- Restart history

---

### View Pod Logs
**Command:**
```bash
kubectl logs <pod-name>
```

**Purpose:**
Shows logs generated by the container inside the pod.

**Use Case:**
Used for debugging applications.

---

### Execute Command Inside Pod
**Command:**
```bash
kubectl exec -it <pod-name> -- /bin/bash
```

**Purpose:**
Opens an interactive shell inside the container.

**Use Case:**
Used for troubleshooting and debugging.

---

## 3. Deployment Management Commands
Deployments manage pods and replicas.

### List Deployments
**Command:**
```bash
kubectl get deployments
```

**Purpose:**
Shows all deployment resources in the cluster.

**Expected Output:**
```text
NAME        READY   UP-TO-DATE   AVAILABLE   AGE
my-nginx    1/1     1            1           10m
```

---

### Create Deployment
**Command:**
```bash
kubectl create deployment my-app --image=nginx
```

**Purpose:**
Creates a deployment running the NGINX container image.

---

### Scale Deployment
**Command:**
```bash
kubectl scale deployment my-app --replicas=3
```

**Purpose:**
Increases or decreases the number of running pods.

**Expected Result:**
Kubernetes creates additional pods.

---

### Delete Deployment
**Command:**
```bash
kubectl delete deployment my-app
```

**Purpose:**
Removes the deployment and its associated pods.

---

## 4. Service Management Commands
Services expose applications running inside pods.

### List Services
**Command:**
```bash
kubectl get services
```

**Purpose:**
Shows all services running in the cluster.

**Expected Output:**
```text
NAME         TYPE           CLUSTER-IP     PORT(S)
my-nginx     LoadBalancer   10.96.1.21     80/TCP
```

---

### Create Service
**Command:**
```bash
kubectl expose deployment my-app --port=80 --type=LoadBalancer
```

**Purpose:**
Creates a service that exposes the deployment to network traffic.

---

### Describe Service
**Command:**
```bash
kubectl describe service my-app
```

**Purpose:**
Displays detailed configuration of the service.

---

## 5. Namespace Commands
Namespaces logically isolate resources in Kubernetes.

### List Namespaces
**Command:**
```bash
kubectl get namespaces
```

**Expected Output:**
```text
NAME              STATUS   AGE
default           Active   10d
kube-system       Active   10d
kube-public       Active   10d
```

---

### Create Namespace
**Command:**
```bash
kubectl create namespace dev
```

**Purpose:**
Creates a new namespace.

---

### Use Specific Namespace
**Command:**
```bash
kubectl get pods -n dev
```

**Purpose:**
Displays pods in a specific namespace.

---

## 6. Resource Monitoring Commands
These commands help monitor cluster resources.

### Check Resource Usage
**Command:**
```bash
kubectl top nodes
```

**Purpose:**
Displays CPU and memory usage of cluster nodes.

**Command:**
```bash
kubectl top pods
```

**Purpose:**
Shows resource consumption of running pods.

---

## 7. Configuration File Commands
These commands work with YAML configuration files.

### Apply Configuration
**Command:**
```bash
kubectl apply -f deployment.yaml
```

**Purpose:**
Creates or updates resources defined in the YAML file.

---

### Delete Configuration
**Command:**
```bash
kubectl delete -f deployment.yaml
```

**Purpose:**
Deletes resources defined in the file.

---

## 8. Troubleshooting Commands
These commands help diagnose cluster issues.

### Get All Resources
**Command:**
```bash
kubectl get all
```

**Purpose:**
Lists most common resources like:
- Pods
- Services
- Deployments
- Replica sets

---

### Watch Resource Changes
**Command:**
```bash
kubectl get pods -w
```

**Purpose:**
Continuously watches changes in pods.

---

## 9. Minikube Commands
When using Minikube.

### Start Cluster
**Command:**
```bash
minikube start
```
**Purpose:** Starts a local Kubernetes cluster.

### Stop Cluster
**Command:**
```bash
minikube stop
```
**Purpose:** Stops the running cluster.

### Open Kubernetes Dashboard
**Command:**
```bash
minikube dashboard
```
**Purpose:** Launches the web UI for monitoring the cluster.
