# Kubernetes Practice Documentation
## Deploying and Running NGINX in Kubernetes

This guide demonstrates how to deploy and expose an NGINX web server in a Kubernetes cluster using Kubernetes, `kubectl`, and Minikube.

---

### 1. Create Deployment
**Command:**
```bash
kubectl create deployment my-nginx --image=nginx
```

**Purpose:**
Creates a Deployment resource in Kubernetes that manages a pod running the NGINX container image.

A Deployment ensures:
- Pods are automatically created
- Pods restart if they fail
- Future scaling and updates are possible

**What Happens Internally:**
1. Kubernetes creates a Deployment object
2. The Deployment creates a ReplicaSet
3. The ReplicaSet creates Pods
4. Each Pod pulls the NGINX container image and runs it

**Expected Output:**
```text
deployment.apps/my-nginx created
```

---

### 2. Open Kubernetes Dashboard
**Command:**
```bash
minikube dashboard
```

**Purpose:**
Opens the web-based Kubernetes dashboard provided by Minikube.

This dashboard allows you to visually monitor:
- Deployments
- Pods
- Services
- Resource usage

**Expected Result:**
- A browser tab opens automatically.
- You will see the Kubernetes cluster interface.
- Your `my-nginx` deployment will appear under **Workloads**.

---

### 3. Check Deployment Status
**Command:**
```bash
kubectl get deployments
```

**Purpose:**
Lists all Deployment resources in the current namespace. This helps verify that the deployment was created successfully.

**Expected Output (Example):**
```text
NAME       READY   UP-TO-DATE   AVAILABLE   AGE
my-nginx   1/1     1            1           1m
```

**Field Explanation:**

| Field | Meaning |
| :--- | :--- |
| **NAME** | Deployment name |
| **READY** | Number of running pods |
| **UP-TO-DATE** | Pods updated to the latest configuration |
| **AVAILABLE** | Pods available to serve traffic |
| **AGE** | Time since deployment was created |

---

### 4. Check Running Pods
**Command:**
```bash
kubectl get pods
```

**Purpose:**
Lists all Pods running in the cluster. Pods are the smallest deployable unit in Kubernetes and contain containers.

**Expected Output:**
```text
NAME                        READY   STATUS    RESTARTS   AGE
my-nginx-6d4cf56db6-abc12   1/1     Running   0          2m
```

**Field Explanation:**

| Field | Meaning |
| :--- | :--- |
| **NAME** | Pod identifier generated by Kubernetes |
| **READY** | Containers ready in the pod |
| **STATUS** | Pod state (Running, Pending, CrashLoopBackOff, etc.) |
| **RESTARTS** | Number of container restarts |
| **AGE** | Time since pod creation |

> **Note:** If `STATUS = Running`, the NGINX container is working correctly.

---

### 5. Expose the Deployment
**Command:**
```bash
kubectl expose deployment my-nginx --port=80 --type=LoadBalancer
```

**Purpose:**
Creates a Service that exposes the deployment to network traffic. Since pods are temporary and dynamic, Kubernetes uses Services to provide stable access.

**Parameters:**

| Parameter | Meaning |
| :--- | :--- |
| `deployment my-nginx` | Target deployment |
| `--port=80` | Port exposed by the service |
| `--type=LoadBalancer` | Exposes the service externally |

**Expected Output:**
```text
service/my-nginx exposed
```

**What Happens Internally:**
Kubernetes creates a Service object that:
- Routes traffic to the NGINX pods
- Provides a stable network endpoint

---

### 6. Verify the Service
**Command:**
```bash
kubectl get services
```

**Purpose:**
Lists all services in the cluster. This helps confirm that the NGINX service is running.

**Expected Output:**
```text
NAME         TYPE           CLUSTER-IP      EXTERNAL-IP   PORT(S)        AGE
my-nginx     LoadBalancer   10.104.34.21    <pending>     80:31234/TCP   1m
kubernetes   ClusterIP      10.96.0.1       <none>        443/TCP        10m
```

**Field Explanation:**

| Field | Meaning |
| :--- | :--- |
| **NAME** | Service name |
| **TYPE** | Service type |
| **CLUSTER-IP** | Internal cluster IP |
| **EXTERNAL-IP** | External access IP |
| **PORT(S)** | Port mapping |

---

### 7. Access the Application
**Command:**
```bash
minikube service my-nginx
```

**Purpose:**
In Minikube, this command creates a temporary tunnel and opens the service in a browser.

**What Happens:**
1. Minikube finds the service
2. Creates a network route
3. Opens the service URL automatically

**Expected Result:**
- Browser opens
- Default NGINX Welcome Page appears
- This confirms that the container is successfully running and exposed.

## Key Kubernetes Concepts Learned

| Concept | Description |
| :--- | :--- |
| **Deployment** | Manages application pods |
| **Pod** | Smallest deployable unit |
| **Container** | Runs the application |
| **Service** | Provides stable network access |
| **LoadBalancer** | Exposes service externally |
